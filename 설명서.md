### 0.4 Mental model — 핵심 컨셉 7개 (pivot 반영)

1. **MPM 기본 사이클** — P2G → grid update → G2P
2. **Cloth XPBD 기본 사이클** — predict → solve constraints → update positions/velocities
3. **두 사이클을 한 frame에 어떻게 끼워 넣을지** — substep, 그리고 transfer phase는 cloth solve 사이에 분리
4. **JKR pull-off energy** — 입자 반경 R, 표면 에너지 γ, 천 reduced modulus K 의 closed-form. 이걸 *force*가 아니라 *event predicate*로 사용
5. **Material-frame σ field** — per-triangle σ + per-vertex p_σ, double-sided. UV에 stable
6. **Mass + linear momentum dual conservation** — attach/detach 시 둘 다 transfer
7. **Hysteresis from JKR curve** — jump-to-contact과 pull-off radius 차이에서 자연 발생, boolean threshold가 아님

### 0.5 Pivot 이전 안과의 차이 (참고)

| 항목 | 원본 W2 안 (deprecated) | W2-pivot (현 안) |
|---|---|---|
| Stick criterion | Heuristic $P_0\cdot\mathbb{1}[v_n<v_{th}]\cdot\exp(-d/\lambda)$ | JKR pull-off energy vs impact KE event predicate |
| Release | Poisson sample on acceleration | Maugis-Dugdale derived hysteretic detach |
| σ representation | Per-vertex world-space scalar | Per-triangle, double-sided, material-frame UV |
| Momentum | 무시 | $\mathbf{p}_\sigma$ pair, conservation 검증 |
| Determinism claim | "deterministic" | "deterministic sampling, aggregate reproducibility" |
| Demos | (a)~(e) 주로 시각적 | 같은 (a)~(e) + 4 ablation baseline + (f) swing-period 정량 |

### 5.2 JKR pull-off work — closed form

```python
@ti.func
def jkr_pulloff_work(radius: ti.f32, gamma: ti.f32, K_reduced: ti.f32) -> ti.f32:
    # W_adh = (15/8π) · π R² · 2γ · (3π γ R / 4 K)^(1/3)
    #   Johnson 1971; Maugis 1992 regularized form
    factor = ti.pow(3.0 * 3.14159 * gamma * radius / (4.0 * K_reduced), 1.0/3.0)
    return (15.0 / (8.0 * 3.14159)) * 3.14159 * radius * radius * 2.0 * gamma * factor

@ti.func
def gamma_humid(gamma_0: ti.f32, beta: ti.f32, h: ti.f32, sigma_local: ti.f32, sigma_max: ti.f32) -> ti.f32:
    # Capillary saturation: γ는 humidity h ∈ [0,1] 와 saturation 의존
    return gamma_0 + beta * h * (1.0 - sigma_local / sigma_max)
```

### 5.3 Attach phase — sort-then-commit (deterministic ordering 가능)

```python
@ti.kernel
def attach_phase_propose():
    """ 매 substep, attach event 후보를 list로 모은다 — commit은 별도 """
    for i in range(N_PARTICLES):
        if particles[i].active == 0: continue

        # closest triangle (broad phase BVH or spatial hash)
        tri_id, dist, bary, side = closest_triangle(particles[i].pos)
        if dist > CONTACT_RADIUS: continue

        # JKR phase-transition criterion
        v_n = compute_normal_velocity(particles[i], tri_id, side)
        E_kin_n = 0.5 * particles[i].mass * v_n * v_n
        sigma_local = sigma_of(tri_id, side)
        gamma_T = gamma_humid(GAMMA_0, BETA, HUMIDITY, sigma_local, SIGMA_MAX)
        W_adh = jkr_pulloff_work(particles[i].radius, gamma_T, K_REDUCED)

        if E_kin_n < W_adh * ti.exp(-dist / LAMBDA):
            # 후보 확정: list에 push (ordered by particle id for determinism)
            attach_events.push(i, tri_id, bary, side)

@ti.kernel
def attach_phase_commit():
    """ Sort된 event list를 한 번에 commit """
    for e in attach_events:
        i, tri_id, bary, side = e
        if particles[i].active == 0: continue   # already taken (race-safe)
        delta_m = particles[i].mass
        delta_p = particles[i].mass * particles[i].vel
        particles[i].active = 0

        # σ 분배 (front/back side에 따라)
        if side == FRONT:
            ti.atomic_add(triangles[tri_id].sigma_front, delta_m)
        else:
            ti.atomic_add(triangles[tri_id].sigma_back, delta_m)

        # p_σ 분배 (3 vertex barycentric)
        v0, v1, v2 = triangles[tri_id].vidx
        ti.atomic_add(vertices[v0].p_sigma, delta_p * bary[0])
        ti.atomic_add(vertices[v1].p_sigma, delta_p * bary[1])
        ti.atomic_add(vertices[v2].p_sigma, delta_p * bary[2])

        # Audit
        ti.atomic_add(mass_attached_this_frame, delta_m)
```

### 5.4 Determinism

- 진짜 bitwise determinism이 필요한 demo는 *segmented reduction* 활성화 (성능 -20% 추정)

### 7.1 Effective mass 갱신

@ti.kernel
def update_effective_inv_mass():
    for v in range(N_VERTICES):
        if vertices[v].fixed:
            vertices[v].inv_mass = 0.0
        else:
            sigma_total = 0.0
            for T in triangles_of(v):
                # Vertex가 triangle T에 기여하는 weight w_{v,T}
                w_vT = barycentric_weight(v, T)
                sigma_total += (triangles[T].sigma_front + triangles[T].sigma_back) * w_vT
            total = vertices[v].mass + sigma_total
            vertices[v].inv_mass = 1.0 / total

@ti.kernel
def reconcile_velocity_with_p_sigma():
    """ p_σ는 vertex velocity에 흡수: v_eff = (m·v + p_σ) / (m + σ) """
    for v in range(N_VERTICES):
        if vertices[v].fixed: continue
        m_eff = 1.0 / vertices[v].inv_mass
        vertices[v].vel = (vertices[v].mass * vertices[v].vel + vertices[v].p_sigma) / m_eff
        vertices[v].p_sigma = ti.Vector([0.0, 0.0, 0.0])  # 흡수 후 reset

## Step 8 — Operator B: Hysteretic Detach + Re-injection (Week 8)

### 8.1 Pull-off work — Maugis-Dugdale에서 도출된 hysteresis

```python
@ti.func
def maugis_pulloff_work(radius: ti.f32, gamma: ti.f32, K_reduced: ti.f32, lambda_M: ti.f32) -> ti.f32:
    # Maugis-Dugdale: jump-to-contact radius vs pull-off necking radius 의 surface energy 적분
    # lambda_M ∈ (0, ∞) 가 Tabor parameter — JKR limit (lambda→∞), DMT limit (lambda→0)
    base = jkr_pulloff_work(radius, gamma, K_reduced)
    return base * (1.0 + 0.3 * lambda_M / (1.0 + lambda_M))  # 정확한 form은 implementation에서 확정
```

이렇게 하면 **stick threshold (jump-to-contact)와 detach threshold (pull-off necking)가 자연 배수로 hysteresis 발생** — boolean 두 개 세팅이 아님.

### 8.2 Detach phase

```python
@ti.kernel
def detach_phase_propose():
    """ Per-triangle 단위로 release 후보를 list로 """
    for T in range(N_TRIANGLES):
        for side in [FRONT, BACK]:
            sigma_T = triangles[T].sigma_front if side==FRONT else triangles[T].sigma_back
            if sigma_T < EPS_SIGMA: continue

            # Inertial work over recent window (vertex 평균 acceleration)
            v0, v1, v2 = triangles[T].vidx
            a_local = (vertices[v0].acc + vertices[v1].acc + vertices[v2].acc) / 3.0
            W_inertial = a_local.norm() * dt_window

            gamma_T = gamma_humid(GAMMA_0, BETA, HUMIDITY, sigma_T, SIGMA_MAX)
            R_grain = MASS_PER_PARTICLE_TO_RADIUS_TABLE[sigma_T]   # 또는 fixed
            W_pulloff = maugis_pulloff_work(R_grain, gamma_T, K_REDUCED, LAMBDA_TABOR)

            if W_inertial > W_pulloff:
                # 몇 입자 release? Inertial excess에 비례
                n_release_estimate = ti.floor(sigma_T / MASS_PER_PARTICLE * (W_inertial / W_pulloff - 1.0))
                n_release_clamped = ti.min(n_release_estimate, ti.floor(sigma_T / MASS_PER_PARTICLE))
                detach_events.push(T, side, n_release_clamped)

@ti.kernel
def detach_phase_commit():
    for e in detach_events:
        T, side, n_release = e
        for _ in range(n_release):
            # Atomic slot allocation (prefix-sum, NOT scan)
            slot = atomic_slot_alloc()
            if slot < 0: break  # buffer 가득

            # Spawn particle
            v0, v1, v2 = triangles[T].vidx
            center = (vertices[v0].pos + vertices[v1].pos + vertices[v2].pos) / 3.0
            normal = triangle_normal(T) * (1.0 if side==FRONT else -1.0)
            particles[slot].active = 1
            particles[slot].pos    = center + normal * SPAWN_OFFSET
            particles[slot].mass   = MASS_PER_PARTICLE
            particles[slot].radius = R_DEFAULT

            # Velocity inheritance: cloth velocity + 작은 outward + p_σ contribution
            avg_vel = (vertices[v0].vel + vertices[v1].vel + vertices[v2].vel) / 3.0
            avg_p_sigma = (vertices[v0].p_sigma + vertices[v1].p_sigma + vertices[v2].p_sigma) / 3.0
            particles[slot].vel    = avg_vel + avg_p_sigma / MASS_PER_PARTICLE * 0.3 + normal * SMALL_KICK

            # σ에서 차감
            if side == FRONT:
                ti.atomic_sub(triangles[T].sigma_front, MASS_PER_PARTICLE)
            else:
                ti.atomic_sub(triangles[T].sigma_back, MASS_PER_PARTICLE)

            # p_σ에서 차감 (proportional)
            ti.atomic_sub(vertices[v0].p_sigma, avg_p_sigma * 0.333)
            ti.atomic_sub(vertices[v1].p_sigma, avg_p_sigma * 0.333)
            ti.atomic_sub(vertices[v2].p_sigma, avg_p_sigma * 0.333)

            # Audit
            ti.atomic_add(mass_released_this_frame, MASS_PER_PARTICLE)
```

### 8.3 Inactive slot 관리 (제대로)

- Naive scan (원 안)은 1M particle에서 비현실적 — **prefix-sum 기반 free-list** 사용
- `atomic_slot_alloc()`이 atomic counter + free-stack 구조로 동작 → race-free, deterministic

### 8.4 검증

- Demo (c): 천 거꾸로 매달기 → release 거의 0 (inertial work < pull-off)
- Demo (d): 강한 shake → release count가 hysteresis curve의 *예측치*와 ±10% 일치
- 떨어진 모래가 다시 모래밭에 합류 (visually 연속)
- σ ≥ 0 항상 유지 (clamp invariant)

## Step 9 — Mass + Linear Momentum Conservation 검증 + Theorem Proof (Week 9)

### 9.1 매 step 검증 (둘 다)

```python
@ti.kernel
def total_mass() -> ti.f64:    # double precision for audit
    total = 0.0
    for i in range(N_PARTICLES):
        if particles[i].active:
            ti.atomic_add(total, ti.cast(particles[i].mass, ti.f64))
    for T in range(N_TRIANGLES):
        ti.atomic_add(total, ti.cast(triangles[T].sigma_front + triangles[T].sigma_back, ti.f64))
    return total

@ti.kernel
def total_linear_momentum() -> ti.types.vector(3, ti.f64):
    total = ti.Vector([0.0, 0.0, 0.0], dt=ti.f64)
    for i in range(N_PARTICLES):
        if particles[i].active:
            ti.atomic_add(total, ti.cast(particles[i].mass * particles[i].vel, ti.f64))
    for v in range(N_VERTICES):
        if not vertices[v].fixed:
            ti.atomic_add(total, ti.cast(vertices[v].mass * vertices[v].vel + vertices[v].p_sigma, ti.f64))
    return total

# main loop
M0 = total_mass()
P0 = total_linear_momentum()
F_ext_integrated = ti.Vector([0.0, 0.0, 0.0], dt=ti.f64)
for frame in range(N_FRAMES):
    step(dt)
    F_ext_integrated += external_force_total() * dt
    M_now = total_mass()
    P_now = total_linear_momentum()
    err_M = abs(M_now - M0) / M0
    err_P = (P_now - P0 - F_ext_integrated).norm() / (P0.norm() + 1e-9)
    assert err_M < 1e-2, f"Mass drift broken at frame {frame}: {err_M}"
    assert err_P < 1e-2, f"Momentum drift broken at frame {frame}: {err_P}"
```

### 9.2 Conservation theorem proof

`paper/proofs/conservation.tex` 에 작성:

- **Lemma 1**: Operator A의 sort-then-commit 구조에서 mass 변화는 transfer 외 0
- **Lemma 2**: Linear momentum도 같은 구조에서 transfer 외 0
- **Lemma 3**: Operator B 대칭으로 같음
- **Theorem 1**: Operator A ∪ B가 frame당 적용되면 invariant 유지

각 step에서 *sort*가 deterministic ordering을 보장한다는 가정 하의 증명. 가정이 깨지면 ULP 비결정성 추가.

### 9.3 누설 발견 시

- atomic_add/sub 누락 → audit log 추적
- 매 step `total_mass()` / `total_momentum()` 호출해 위반 step 찾기
- Float 누적 오차면 double 누적 + single 변환

## 자주 발생하는 문제 (FAQ)

### Q. Taichi가 GPU를 못 찾음
- WSL2: `nvidia-smi` 동작 확인. 안 되면 Step 0.2.
- Native Linux: `nvidia-modprobe` 설치, `/dev/nvidia*` 권한 확인.

### Q. Mass conservation이 1%를 넘음
- atomic_add/sub 누락 가능성 → 모든 transfer 경로 추적
- 매 step `total_mass()` / `total_linear_momentum()` 호출해 위반 step 찾기
- Float 누적 오차면 double 누적 + single 변환
- Sort-then-commit pipeline 외부에서 σ 수정하지 않았는지 확인

### Q. JKR pull-off work가 너무 stiff (즉시 부착되거나 즉시 detach)
- $K_{\text{reduced}}$ 가 실제 cloth modulus와 sand grain modulus의 reduced form인지 확인
- $\gamma_0$ 가 dry sand-cloth surface energy 정도 (≈ 0.01-0.1 J/m²) 인지
- Maugis Tabor parameter $\lambda_M$ 으로 regularization 조정

### Q. Hysteresis가 단순 boolean threshold처럼 동작
- `maugis_pulloff_work`가 `jkr_pulloff_work`와 *반드시 다른 값*이어야 함
- Tabor parameter $\lambda_M$ 가 0이면 DMT limit (no hysteresis) — sweep으로 확인
- Detach 조건이 `W_inertial > W_pulloff`인지 (`> W_adh`가 아니라)

### Q. σ가 cloth fold에서 swimming
- Material-frame UV가 per-triangle인지 (per-vertex가 아닌지)
- Step 5.2에서 σ를 `triangles[T].sigma_*`에 저장하고 vertex로 *broadcast만* 하는지

### Q. Demo (f) swing period 차이가 안 보임
- `update_effective_inv_mass()` + `reconcile_velocity_with_p_sigma()`가 매 substep 호출되는지
- `vertices[v].inv_mass` 가 XPBD constraint solve에 *실제로* 사용되는지

### Q. Ablation baseline이 우리와 같이 보임
- Demo scene이 *너무* easy일 가능성 — humidity sweep, fold geometry, swing dynamics를 강제로 활성화하는 scene으로 설계
- Quantitative metric (period shift, R² fit, σ swimming distance)이 visual 차이를 보완

### Q. RTX 4080 16GB로 메모리 부족
- MAX_N (입자 수) 축소: 1M → 500k → 250k tier
- MPM grid 해상도: 128³ → 96³ → 64³
- σ 자료구조 비용: per-triangle f32 × 2 (front/back) = 5k triangle × 8B = 40 KB (negligible)
- p_σ: per-vertex vec3 f32 = 5k vertex × 12B = 60 KB (negligible)

### Q. 5개 demo 중 1~2개만 plausible
- 시나리오별 parameter set을 별도로 (`data/configs/demo_*.yaml`)
- 한 set으로 모든 시나리오 만족시키려 하지 말고, 학기 프로젝트는 시나리오별 튜닝 OK
- 단, *ablation*은 동일 시나리오에서 같은 파라미터 — baseline 비교 위해
